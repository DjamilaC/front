=>Le 07_bis_float =>Notion de flux

le flux d'un document pourrait se definir comme étant le comprtement naturel d'affichage des éléments d'une page web. autrement dit, les elèments succedent dans l'ordre ou ils sont déclarés dans le code HTML. de façon simpliste, les elements de séquençage tels que "he", "p", ou "ul", s'affichent logiquement pae défaut les uns en dessous des autres tandis que les elements délimitant pour la plupart des portions de textes( mise en étalique d'un ou plusieurs mots par exemple) s'affichent cote à cote
maintenant que nous avons evoqué ce point, il faut comprendre que les balises HTML ont toutes par defaut des propriétes de rendu CSS particulières. en fait, il existeinitialement deus grands groupes de balises: <les balises de rendu CSS "bloc"(bloc) les balises de reudu CSS "en-ligne"("inline") ces valeurs de rendu visuel coincident généralement par defaut avec le groupe d'appartenance HTML

Des proprietés CSS d'un élement(dusplay:block, inline, list-litem, inline-block, table, table-cell,...) découlent des spécifiés d'affichage, son positionnement dabs le flux.
il est possible de sortir du flux naturel un ou plusieurs elementspour modifier leur affichage avec du CSS

Nos deux groupes d'elements peuvent etre classées en deux Les élements de type block(h1,p,ul,ol,dl,table,blockquote etc,...)les eléments de type inline(a,img,strong,abbr, etc...)

=> Le type "block"
les elemnts de rendu CSS block se placent toujours en dessousde l'autre par defaut(comme un retour à la ligne.)par exemple uen suite de paragraphe
  • il occupe lentiereté de la largeur de son conteneur
  • il permet l'attribution de marges verticales
  • il permet la modification de sa hauteur et de sa largeur
le premier point est primordial, nous utilisons abondamment ce comportement par défaut bien pratique dans la création de nos mise en page. notons également que tout élément peut étre "reclassé" dans la famille opposée grace à la propriété display( que nous verrons plus tard).
